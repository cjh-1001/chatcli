Findings

critical Process Injection Capability sub_140001200

The binary allocates memory in a target process (VirtualAlloc), writes shellcode, and creates a remote thread to execute it. This is a definitive indicator of code injection.

VirtualAlloc(hProcess, NULL, dwSize, MEM_COMMIT, PAGE_EXECUTE_READWRITE);
WriteProcessMemory(hProcess, lpBaseAddress, lpBuffer, dwSize, NULL);
CreateRemoteThread(hProcess, NULL, 0, lpStartAddress, NULL, 0, NULL);
🔧 Remediation:

Block CreateRemoteThread calls from non-whitelisted processes via EDR policy. Monitor for VirtualAlloc with PAGE_EXECUTE_READWRITE permissions.

high Registry Persistence sub_140001800

Creates a Run key to ensure execution on every system boot.

RegCreateKeyExW(HKEY_CURRENT_USER, "Software\\Microsoft\\Windows\\CurrentVersion\\Run", ...)
RegSetValueExW(hKey, L"SecurityHealth", ...)
🔧 Remediation:

Monitor registry Run key modifications. Use AppLocker or WDAC to restrict unauthorized executables.

medium Suspicious String Obfuscation .rdata section

Multiple XOR-encoded strings found. The decoding routine at sub_140001500 uses a single-byte XOR key (0x55).

Key: 0x55
Decoded: "C:\\Users\\Public\\svchost.exe"

🔧 Remediation:

Use static string deobfuscation tools to detect XOR-encoded indicators.
